3. Personal Data We Collect

To provide the Application's services, we process the following categories of data:

3.1 Registration and Profile Data

  • Email address
  • Name or public alias
  • Date of birth and age
  • Gender and gender identity
  • Profile photos and private photos (stored on Cloudinary)
  • Voice introduction audio recordings (stored on Cloudinary)
  • Biography or personal description
  • Search preferences (gender of interest, type of relationship sought, age range and maximum distance)
  • Interests and hobbies

3.2 Location Data

  • Geographic coordinates (latitude and longitude), obtained from the device with the user's express consent
  • Approximate city or locality, derived by reverse geocoding of the coordinates

3.3 Activity and Usage Data

  • Date and time of last connection (lastActive)
  • In-app interactions: likes and matches
  • Messages exchanged between mutually matched users

3.4 Technical and Device Data

  • Push notification token (Firebase Cloud Messaging)
  • Unique user identifier generated by the authentication system

3.5 Third-Party Sign-In Data

If the user chooses to sign in via Google Sign-In, we will receive the user's public name, email address and Google account identifier, in accordance with Google LLC's privacy policy.

3.6 Payment Data

For in-app purchases and subscriptions on the web version, payments are processed by Stripe. The Controller does not store card data at any point; Stripe handles all payment data in accordance with PCI DSS standards. For purchases on the mobile app, billing is managed by Google Play and RevenueCat. We receive only confirmation of purchase status and subscription state — not card details.

3.7 Emergency Contact Data (SOS Function)

If the user sets up an emergency contact, we will process the name, phone number and, optionally, a personalised message for that contact, provided voluntarily by the user. The user guarantees that they have obtained the emergency contact's consent to share this data with the Application.

When the SOS function is activated, the user's exact GPS location at that moment, together with the configured message, is sent by SMS to the indicated phone number. Unlike the rest of the Application (see section 13), in this case the exact location is shared, exclusively with the contact designated by the user and only at the moment the user voluntarily activates this safety function.

12. Push Notifications

The Application may send push notifications to the user's device to inform them of new matches, received messages and other relevant account activity.

Sending push notifications requires the user's express consent, which is requested during installation or first use of the Application. The user may manage or revoke this permission at any time from their Android device's notification settings.

For the technical management of notifications, we use Google's Firebase Cloud Messaging (FCM) service. The notification token generated by the device is stored in the Application's database exclusively for this purpose.

13. Geographic Location

The Application accesses the device's location in the foreground for the following purposes:

  • Showing the user profiles of other registered users in their geographic area.
  • Calculating and displaying the approximate distance between users.
  • Displaying the city or locality associated with the user's profile.

The Application does not access location in the background when it is not active.

The user's exact coordinates are never visible to other Application users; only the approximate distance (in kilometres) and the city or locality are shown, in order to protect each person's privacy.

Exception — SOS function: if the user voluntarily activates the emergency button, their exact location at that moment is shared by SMS with the emergency contact the user has configured (see section 3.7). This is the only circumstance in which the exact location is shared, and it only occurs as a deliberate action by the user in an emergency situation.

The location permission may be revoked at any time from device settings. Revocation will not prevent general use of the Application, but will limit the proximity search functionality.

14. Content Moderation

In order to ensure a safe and respectful environment for all users, the Application subjects profile photos to a process of automated moderation via the Sightengine API.

This process analyses images to detect the presence of explicit, violent, discriminatory or community-standards-violating content, before such images are published and visible to other users.

Images are transmitted to Sightengine solely for this specific analysis and are not stored by that provider after processing. The result of the analysis (approval or rejection) is processed by the Application automatically.

In addition to automated moderation, the Controller reserves the right to manually review content reported by users as inappropriate, and to take appropriate measures (content removal, account suspension or cancellation) in the event of a breach of the terms of use.
